Scaling the "Cecil" Initiative at CX By Design

Leading 3 cross-functional intern teams to build a full client experience system from the ground up.

Context

CX By Design needed a structured internal system to manage client prospecting, awareness, and onboarding — but had no repeatable workflow or CRM process in place.

Objective

Build scalable client journey workflows, a functioning CRM process using HubSpot, and measurable content frameworks — all managed through distributed intern teams.

Challenge

Managing 3 teams working across prospecting, awareness, and CX systems simultaneously — with no prior playbooks, unclear ownership, and tight timelines.

Solution

Impact

Measurable improvements in team coordination, stakeholder visibility, and client enablement — with process documentation enabling future team ramp-up in days, not weeks.

Client Success 2025

Reducing Support Tickets 30% at Circler.io

Redesigning the client onboarding experience to cut friction, automate responses, and improve product adoption.

Context

Circler.io — a NYC-based startup — was seeing high volumes of repetitive client support tickets post-onboarding, straining the team and slowing product growth.

Objective

Reduce repetitive client support load while improving the new client onboarding experience and conversion from demo to trial.

Challenge

No centralized knowledge base existed. Client friction points were invisible — buried in ticket history and informal Slack threads. No automated communication workflow was in place.

Process

Used Hotjar and Google Analytics to map client behavior and identify drop-off points in the onboarding flow. Audited 3 months of support tickets to cluster recurring questions. Collaborated with marketing on messaging gaps.

Solution

Built a centralized FAQ hub, automated welcome campaigns, recorded demo walkthroughs, and created interactive onboarding guides. Partnered with marketing to revamp CTAs on the product website.

Impact

Repetitive support tickets dropped 30%. Demo-to-trial conversion improved. New client onboarding became a repeatable, documented process rather than a manual effort each time.

30% Reduction in support tickets
Hotjar Behavior analytics used
Jira Release tracking & UAT
Product / AI NYCE Challenge · 2024

AidNYC — AI-Powered Flood Alert Platform

Competing in NYC's AI Innovation Challenge to build a real-time public safety tool using ML and geospatial data.

Context

NYC faces recurring flood risks that disproportionately affect lower-income neighborhoods. Existing alert systems lacked real-time neighborhood-level precision.

Objective

Design and deploy a platform that identifies flood-prone areas in real time and navigates affected residents to safety using geospatial data points.

Challenge

Evaluating multiple pre-trained ML models for accuracy under time and compute constraints. Ensuring the API layer could handle real-time, high-volume city-scale queries.

Process

Tested and benchmarked several pre-trained ML models using IBM Watson Governance. Selected the highest-accuracy model, then optimized the API pipeline for performance and scalability.

Solution

A scalable, API-driven platform that ingests data points, identifies flood risk zones, and navigates individuals in real time. Deployed with several pipeline optimizations for throughput.

Impact

Selected for the NYCE AI Innovation Challenge. Demonstrated end-to-end ownership from problem framing through ML evaluation and API deployment.

IBM Watson Governance used
NYCE AI Challenge finalist
NYC-scale Real-time API delivery
